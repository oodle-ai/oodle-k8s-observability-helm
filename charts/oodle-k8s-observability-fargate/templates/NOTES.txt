=== Metrics ===

Metrics are collected automatically. No changes to your workloads needed.

  - Cluster-level: deployment replicas, pod phases, node conditions, HPA status
  - Pod/container: CPU, memory, disk, network for ALL pods across ALL Fargate nodes
    (scraped via Kubernetes API server proxy to each node's cadvisor)

=== Logs ===

Fargate's built-in Fluent Bit sends all container logs to Oodle automatically.
No sidecars, no DaemonSets — every pod in every namespace is covered.

  Host:  {{ include "oodle-fargate.logsHost" . }}

  Pre-requisite: The Fargate Pod Execution Role needs outbound HTTPS
  access to {{ include "oodle-fargate.logsHost" . }}.

  Restart existing pods after first install to pick up the log router config.

=== Traces (Java Auto-Instrumentation) ===

Gateway OTLP endpoint (for your app env vars):
  {{ include "oodle-fargate.gatewayOtlpEndpoint" . }}

Step 1: Bake the OTel Java agent into your Docker image (one-time):

    # Add to your Dockerfile
    ADD https://github.com/open-telemetry/opentelemetry-java-instrumentation/releases/latest/download/opentelemetry-javaagent.jar /opt/opentelemetry-javaagent.jar
    ENV JAVA_TOOL_OPTIONS="-javaagent:/opt/opentelemetry-javaagent.jar"

Step 2: Set the gateway endpoint in your Kubernetes Deployment:

    env:
      - name: OTEL_EXPORTER_OTLP_ENDPOINT
        value: "{{ include "oodle-fargate.gatewayOtlpEndpoint" . }}"
      - name: OTEL_SERVICE_NAME
        value: "<your-service-name>"

  No API keys needed in your app — the gateway handles authentication.
  Works from any namespace. Just set the env vars and deploy.
{{- end }}
